## 1. 产品概述

### 1.1 一句话描述

**IntervYOU** 是一个 AI 驱动的面试教练，从简历出发，通过动态追问模拟真实面试，并提供可操作的改进建议——不只是模拟面试，而是帮你把简历上的每一句话都变成能接住追问的弹药。

### 1.2 产品愿景

让每一个校招生在第一次真正面试之前，已经"面过"100 次。

### 1.3 核心价值主张

| 传统方案 | IntervYOU |
|---------|-----------|
| 固定题库，问完就没了 | 基于简历动态生成问题，无限追问 |
| 评分模糊（"75分"） | 逐题精细点评 + 具体改进示例 |
| 简历和面试割裂 | 简历 → 追问 → 复盘 闭环 |
| 不敢开口，没有练习机会 | 零后果、可重复、多难度的安全练习环境 |

## 2. 市场背景与竞品分析

### 2.1 市场现状

- 2025 年中国高校毕业生 1222 万人，校招竞争白热化
- AI 面试已在大厂广泛使用（快消/互联网行业的 AI 初筛）
- 但 **AI 辅助练习** 的产品仍处于早期，头部产品体验粗糙

### 2.2 竞品地图

| 产品 | 类型 | 优点 | 痛点 |
|------|------|------|------|
| 牛客 AI 模拟面试 | 题库+录音 | 题库量大，有评分 | 缺少个性化追问，反馈笼统 |
| 赛码 AI 面试 | SaaS 平台 | 企业级，结构化 | 面向企业而非求职者 |
| Interviewing.io | 真人模拟 | 反馈专业 | 贵（$100+/次），需预约 |
| Pramp | 真人配对 | 免费 | 匹配质量不稳定 |
| ChatGPT/Claude 通用对话 | AI 对话 | 灵活 | 没有结构化面试流程，不会追问 |
| Yoodli | AI 演讲教练 | 语音分析强 | 面向演讲而非面试，缺少简历联动 |
| 超级简历/知页 | 简历工具 | 简历优化好 | 没有面试模拟功能 |

### 2.3 差异化定位

> **"简历 + 面试 + 复盘" 三合一的 AI 面试教练——从源头（简历）到终点（面试表现）的完整闭环。**

核心壁垒：
1. **简历逐条追问引擎** — 不是随机出题，而是针对简历中每个项目经历自动生成追问链
2. **动态追问决策** — 根据回答质量实时决定追问深度，而非预设脚本
3. **可操作反馈** — 不只给分数，给出"你的 STAR 缺了 R（Result）"，并提供改写示例
4. **难度梯度设计** — 友好/标准/压力三档，从建立信心到极限施压逐级训练

### 5.1 核心流程

```
┌─────────┐    ┌─────────┐    ┌─────────────┐
│ 上传简历  │───▶│ 选择模式  │───▶│ 选择难度     │

                                     │
                                     ▼
┌─────────┐    ┌─────────┐    ┌─────────────┐
│ 查看报告  │◀───│ 面试结束  │◀───│ 模拟面试     │

└─────────┘    └─────────┘    └─────────────┘
```

### 5.2 面试环节流程（综合模拟模式为例）

```
1. 开场白
   └─ 面试官自我介绍 + 说明面试结构

2. 自我介绍（1轮）
   └─ 候选人自我介绍 → [评估：是否超过2分钟/有无亮点/结构]

3. 简历深挖（3-5轮）
   └─ 针对最有亮点的项目追问
   └─ 追问链示例：
       Q: "你在XX项目中负责了用户增长，具体做了什么？"
       A: "我优化了推荐算法"
       Q: "怎么优化的？你们用的是什么算法？"
       A: "协同过滤"
       Q: "冷启动问题你们怎么解决的？"
       └─ [逐层深入直到答不上来或追问耗尽]

4. 行为问题（2-3轮）
   └─ 从问题库 + 简历适配生成
   └─ 例："讲一个你跟团队成员意见不一致的经历"

5. 情景假设（1-2轮）
   └─ 例："如果产品上线前一天发现关键bug，作为PM你怎么决策？"

6. 反问环节（1轮）
   └─ 候选人反问面试官问题
   └─ [评估反问质量]

7. 结束
   └─ 生成面试报告入口
```

## 6. 功能详细设计

### 6.1 简历上传与解析

**交互流程：**
1. 用户拖拽/点击上传简历文件
2. 系统展示解析进度（"正在解析你的简历…"）
3. 解析完成后展示结构化预览（可编辑）
4. 用户确认 → 存入当前会话

**解析策略（双引擎）：**
- **引擎 1 — 文本提取**：pdf-parse / mammoth（提取原始文本）
- **引擎 2 — AI 结构化**：将原始文本发送给 Claude，返回结构化 JSON

**结构化数据格式：**
```json
{
  "name": "张三",
  "education": [
    { "school": "清华大学", "degree": "硕士", "major": "计算机科学", "year": "2026" }
  ],
  "experiences": [
    {
      "type": "internship",
      "company": "字节跳动",
      "role": "AI产品实习生",
      "duration": "2025.06 - 2025.09",
      "description": "负责XX推荐系统的产品迭代，DAU提升15%",
      "highlights": ["DAU提升15%", "从0到1搭建AB实验平台"],
      "tech_stack": ["推荐系统", "AB测试", "SQL"]
    }
  ],
  "skills": ["Python", "SQL", "产品设计", "数据分析", "Axure"],
  "projects": [...]
}
```

### 6.2 面试模式选择

**模式选择界面设计：**

| 模式 | 图标 | 适合场景 | 回合数 | 核心追问策略 |
|------|------|---------|--------|-------------|
| 简历深挖 | 🔍 | 简历写了但怕被追问 | 8-12轮 | 逐项追问，直到挖不出细节 |
| 行为面试 | 💬 | 准备宝洁八大问等 | 6-8轮 | STAR 框架，追问每个环节 |
| 技术面试 | 💻 | 技术岗专业知识 | 8-10轮 | 知识点 + 项目技术选型 |
| 综合模拟 | 🎯 | 全真模拟真实面试 | 12-15轮 | 完整流程，多题型混合 |

### 6.3 难度设计

| 维度 | 😊 友好模式 | 😐 标准模式 | 😤 压力模式 |
|------|-----------|-----------|-----------|
| 语气 | 鼓励、引导 | 专业、中立 | 质疑、挑战 |
| 追问深度 | 1-2层，答不出就提示 | 2-3层，追问到答不出 | 3-5层，持续施压 |
| 提示行为 | 答不出时给引导 | 不提示 | 答不出时质疑 |
| 时间压力 | 无 | 温和提醒 | 明确催促 |
| 评分严格度 | 宽松 | 标准 | 严格 |
| 适用用户 | 面试新手 | 有一定经验 | 冲刺大厂 |

**压力模式示例对话：**

> 🤖：你在字节做的推荐系统优化，DAU 提升了 15%，这个数字是怎么算出来的？
>
> 😊 友好：你可以想想是按周同比还是跟对照组比较的？没关系，慢慢想。
>
> 😐 标准：请具体说明统计口径和对比基准。
>
> 😤 压力：15%——这个数字我听起来有点虚。你能现场拆一下计算逻辑吗？如果我现在让你证明这 15% 确实是你带来的而不是自然波动，你怎么证明？

## 7. 评估体系设计

### 7.1 评分维度

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| **STAR 完整性** | 25% | Situation(背景) → Task(任务) → Action(行动) → Result(结果) 是否完整 |
| **量化能力** | 20% | 是否使用具体数字、百分比、对比数据 |
| **逻辑清晰度** | 20% | 回答结构是否清晰、是否有条理 |
| **技术/专业深度** | 20% | 对项目细节、技术原理的掌握程度 |
| **沟通表达** | 15% | 语言流畅度、自信度、时长控制 |

### 7.2 绿/黄/红 标记规则

| 标记 | 条件 | 含义 |
|------|------|------|
| 🟢 良好 | 得分 ≥ 4/5 | 回答扎实，继续保持 |
| 🟡 一般 | 得分 2-3/5 | 有亮点但不够，建议改进 |
| 🔴 薄弱 | 得分 ≤ 1/5 | 完全没答到点上，需重点准备 |

**触发 🔴 的典型场景：**
- 简历写了但完全答不出来（"这个项目我其实参与不多"）
- STAR 四个维度缺了三个
- 答非所问，绕了30秒没入正题
- 出现负面信号（"这个不是我做的"、"我不太清楚"连续出现）
