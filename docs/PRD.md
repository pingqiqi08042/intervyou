# IntervYOU — 产品需求文档

> 版本 v2.2 | 石喜斌 | 2026.07

---

## 1. 产品概述

IntervYOU 是一个 AI 面试教练平台，面向校招生提供从简历诊断到模拟面试再到复盘优化的完整训练闭环。核心差异化在于**简历驱动动态追问**——不是随机出题库，而是根据候选人的每一句回答实时生成追问，模拟真实面试官的思维。

已部署上线：https://intervyou-pro.netlify.app

---

## 2. 问题定义

### 2.1 用户痛点

校招生准备面试时面临三个核心问题：

- **缺乏动态追问练习**：市面 AI 面试工具（牛客、赛码）本质是固定题库+录音回放，不管候选人怎么回答，下一个问题都是预设的。真实面试中，面试官会根据回答质量决定追问方向。
- **反馈不可操作**：现有工具给的是"表达能力 75 分"这样的模糊评分，候选人不知道具体哪里扣分、怎么改进。
- **简历和面试割裂**：简历工具帮你改简历，面试工具帮你练面试——但面试官就是从简历出发提问的，简历上写的每一个字都可能被追问。

### 2.2 竞品对比

| 产品 | 类型 | 问题 |
|------|------|------|
| 牛客 AI 模拟面试 | 题库+录音 | 缺少个性化追问，反馈笼统 |
| 赛码 AI 面试 | SaaS 平台 | 面向企业而非求职者 |
| 超级简历/知页 | 简历工具 | 没有面试模拟功能 |

**差异化定位**：简历+面试+复盘三合一，所有问题从简历出发，根据回答质量动态追问。

---

## 3. 目标用户

应届毕业生，求职互联网/科技行业。第一次面试或者面试经验少，不知道自己简历会被追问什么，练习完不知道答得好不好、怎么改进。

## 4. 核心功能

```
IntervYOU
├── 简历管理
│   ├── 文件上传（PDF/Word/Markdown）+ 粘贴文字
│   ├── AI 结构化解析（提取经历、项目、技能）
│   ├── 7 维诊断报告（评分+风险点+改写建议）
│   └── JD 输入（可选，诊断更精准）
│
├── 模拟面试（核心）
│   ├── 5 种岗位自动识别（后端/前端/产品/数据/运营）
│   ├── 4 层递进追问框架
│   ├── 3 种难度模式（引导/标准/压力）
│   ├── 语音输入（Web Speech API）
│   └── 退出/暂存机制
│
├── 复盘报告
│   ├── 五维雷达图（STAR/量化/逻辑/深度/表达）
│   ├── 逐题红/黄/绿点评
│   ├── 优势/改进/简历建议
│   └── 面试回看功能
│
└── 优化简历
    ├── 3 种生成模式（仅诊断/仅面试/综合）
    ├── Markdown + .docx 下载
    └── 选择具体面试记录生成
```

## 5. 核心设计决策

### 5.1 为什么是 4 层递进而非自由追问

自由追问不可控——Agent 容易在同一个点上反复追问，或者跳过关键细节。4 层递进（概述→细节→难点→量化）强制保证每段经历都追到位，同时也给了 Agent 清晰的行为框架。

### 5.2 为什么选 DeepSeek 而非 Claude

Claude 输出更稳定但成本是 DeepSeek 的 50 倍。对于高频面试场景——用户可能每天练习多次——成本差距不可接受。DeepSeek 的 JSON 格式不稳定问题通过四层容错链路兜底。同时保留多模型切换入口，用户可自行选择。

### 5.3 为什么建 Prompt 测试框架

Prompt 调优靠人工感受无法规模化——改了某一句，可能修好一个 case 又破坏另一个。17 个测试用例覆盖开场、追问、换题、压力模式、边界场景，每次改 Prompt 跑一遍，量化对比改进效果。

## 6. 评估体系

| 维度 | 权重 | 说明 |
|------|------|------|
| STAR 完整性 | 25% | Situation-Task-Action-Result 是否完整 |
| 量化能力 | 20% | 数字/百分比/对比基准的使用 |
| 逻辑清晰度 | 20% | 结构是否清晰、层次是否分明 |
| 技术/专业深度 | 20% | 对项目细节的理解深度 |
| 沟通表达 | 15% | 语言流畅度、自信度 |

---

## 7. 技术概要

- **前端**：Next.js 14 + React 18 + Tailwind CSS + Recharts（雷达图）
- **后端**：Next.js API Routes + Prisma ORM
- **数据库**：本地开发 SQLite，生产 Neon PostgreSQL
- **AI 模型**：默认 DeepSeek V3，可切换 Claude / OpenAI
- **部署**：Netlify，自动 CI/CD

---

## 8. 已知局限

- DeepSeek 多轮对话后续注意力衰减，偶发指令遵循偏差（通过服务端重复拦截缓解）
- 语音输入仅支持 Chrome/Edge（Web Speech API 限制）
- 无流式输出，面试回复为全量等待
- 缺少真实用户数据，迭代依据为个人自测而非用户行为分析
