# IntervYOU 项目审计

> 日期：2026-07-11 | 状态：MVP 功能基本完成，需要打磨

---

## 一、项目概况

一个面向校招生的 AI 面试教练平台。核心流程：上传简历 → 诊断分析 → 模拟面试 → 复盘报告 → 优化简历。

**技术栈**：Next.js 14 + Prisma/SQLite + Tailwind CSS + DeepSeek V3（可切换 Claude/OpenAI）

---

## 二、已实现功能 & 完成度评估

### 2.1 简历管理

| 功能 | 状态 | 效果 |
|------|------|------|
| 上传简历（PDF/Word/MD） | ✅ 完成 | 正常，pdf-parse 1.1.1 版本稳定 |

| 功能 | 状态 | 效果 |

| 4 种面试模式 | ✅ 完成 | 简历深挖/行为/技术/综合，行为差异基本可用 |

## 四、当前最大问题（按优先级）

### P0 — 面试体验不稳
- Agent 追问质量波动大：有时深度追问，有时草草换题
- 偶发提前结束（虽然已加多层防护）
- 综合模式阶段切换生硬

### P1 — 诊断评分单调
- 不同简历容易得到相似分数（68-75 区间）
- prompt 已调整但效果待验证

### P2 — UI/UX 粗糙
- 没有加载骨架屏
- 错误提示不够友好（直接显示技术错误信息）
- 移动端未适配
- 缺少空状态引导

### P3 — 缺少关键功能
- 没有真正的 .docx 导出（目前是 HTML 伪装的 .doc）
- 复盘报告没有雷达图
- 没有多份简历对比功能

## 五、建议下一步

### 选项 A：打磨面试质量（推荐）
继续调 Agent Prompt，增加更多面试场景的测试用例，提升追问一致性。这是产品核心价值所在。

### 选项 B：提升 UI/UX
统一错误处理、加骨架屏、移动端适配、交互动画。让产品"看起来像回事"。

### 选项 C：补齐缺失功能
雷达图、真正 .docx、简历对比、面试录音回放分析。

### 选项 D：准备部署上线
配置 Vercel 部署、购买域名、写 README、录 Demo 视频。让项目可以写进简历。

---

> **建议**：A → D → B → C。先把面试质量稳住，然后尽快部署上线让项目可展示，再打磨体验和加功能。
