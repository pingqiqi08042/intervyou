## 二、自我介绍怎么说（60 秒版）

> 我独立做了一个 AI 面试教练项目，叫 IntervYOU。
>
> 做这个项目是因为我发现：校招生准备面试时，最大的痛点不是缺题库，而是没有人给他动态追问。市面上现有的 AI 面试产品本质是固定题库 + 录音回放——不管你怎么回答，下一个问题都是预设的。真正的面试官会根据你的回答动态深挖。
>
> 所以我设计了一套让 AI 能够从简历出发、根据回答质量动态追问的系统。核心是三层设计：第一层是岗位自动识别，根据简历内容判断你是后端还是产品；第二层是四层递进提问——从概述到细节到难点到量化，保证每个项目都挖到底；第三层是防重复机制，Prompt 层面加服务端层面双重拦截。
>
> 整个产品从设计到开发到部署全是我一个人做的，目前已经上线了。

## 三、面试官会问什么 & 怎么回答

### Q1："这个项目的核心创新点是什么？"

> 不是题库驱动的，是简历驱动的动态追问。传统 AI 面试是"问一个问题 → 听回答 → 问下一个预设问题"，我的系统会先解析简历，针对简历中每一段经历，按照四层递进框架追问——概述了解全貌 → 细节深挖技术栈 → 难点追问 trade-off → 量化追问数据结果。每层问完自动推进，不会在同一层打转。这更像真实面试官的思维方式。

### Q2："Prompt 是怎么设计的？"

> 核心思路是模块化 + 可测试。我把 System Prompt 拆成了岗位范式、四层递进框架、三模式差异化、防重复规则、输出格式五个独立模块，由 TypeScript 函数组装。改了某一块不会影响其他模块。
>
> 更重要的是我建了一个测试框架——17 个测试用例覆盖各种边界场景，比如"候选人给模糊回答时 Agent 应该追问具体方向"、"连续 3 轮敷衍应该自动换题"。每次改 Prompt 之后跑一遍测试确认没有退化。

### Q3："为什么选 DeepSeek 而不是 Claude？"

> DeepSeek 便宜 20 倍，中文能力也强，新用户注册送 500 万 tokens。但 DeepSeek 的 JSON 格式偶尔不稳定——我做了四层 Fallback 容错：JSON 解析失败 → 追加纠错消息重试一次 → 从返回文本里提取有用的部分 → 基于上下文生成默认问题。实测下来 JSON 不稳定率从 5% 降到了几乎没有。
>
> 产品设计上我也允许用户切换 Claude——如果追求最好效果可以自己配 Claude Key。

### Q4："追问质量怎么保证的？怎么解决重复提问？"

> 我们最头疼的问题就是 Agent 会重复问"你具体做了什么"。我做了两层防护：
> - Prompt 层：最顶部写了"重复提问 = 面试失败"，要求每轮回复前检查上一轮问了什么
> - 服务端层：如果 Agent 的新回复前 15 个字和之前的消息一样，自动替换为点名具体公司的换题语
>
> 上线后的版本，17 个测试用例里追问重复率为零。

### Q5："这个产品怎么衡量效果？"

> 目前还在早期，主要通过 Prompt 测试框架的通过率和人工评估来衡量 Agent 行为是否符合预期：
> - 17 个测试用例覆盖了开场、追问、换题、压力模式、边界场景
> - 追问重复率从上线初的约 20% 降到零
> - JSON 格式错误率从 5% 降到接近零
>
> 如果继续做，我会上埋点追踪三个指标：单次面试完成率、人均对话轮次（反映追问深度）、7 日回访率（反映用户是否真的在练）。

### Q6："最大的技术挑战是什么？"

> 对话历史污染问题。Agent 每次回复是 JSON 格式，里面包含内部评分（比如 STAR 完整性 3 分）。如果这个 JSON 原样回传给 LLM，它就会看到自己之前的评分——可能导致评分泄露或评分锚定。我的解决方案是在每轮发送给 LLM 之前，解析历史消息的 JSON，只保留 `message` 字段，去掉所有评估数据。这个改进让 Token 消耗降了 53%，也杜绝了评分泄露。

### Q7："你在这个项目中学到了什么？"

> 最大的收获是对 Prompt Engineering 的理解深了很多。以前以为 Prompt 就是写好文字就行，做了这个项目才知道：Prompt 是一个需要测试、需要版本管理、需要容错的工程问题。比如温度参数对 JSON 模式的影响——0.7 太高会导致格式不稳定，0.5 刚好；再比如 Prompt 越长不一定是好事——我把 Prompt 从 3600 tokens 压缩到 1400 tokens，追问质量反而上升了。另外也学到了怎么把 AI 产品从 0 到 1 做出来——不只是想，是真的写代码、测试、部署、上线。

### Q8："如果继续做，下一步做什么？"

> 三个方向：一是做面试过程的数据分析——比如统计哪种岗位的追问深度最浅，反过来优化 Prompt；二是做流式输出，目前是全量等待，改成 SSE 逐字输出体验会好很多；三是引入真实面试录音分析——用户上传录音，AI 逐句分析语速、停顿、口头禅。
