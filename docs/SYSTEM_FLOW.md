# IntervYOU — 系统全流程

> 最后更新：2026-07-11

---

## 总览：两条路径 + 一个闭环

```
                              ┌─────────────────────┐
                              │      🏠 首页         │
                              │  上传简历（唯一入口）  │
                              └──────────┬──────────┘
                                         │
                                  简历文件上传
                                         │
                                         ▼
                              ┌─────────────────────┐
                              │  📄 简历解析          │
                              │  pdf-parse/mammoth   │
                              │  → Claude Haiku 结构化│
                              └──────────┬──────────┘
                                         │
                        ┌────────────────┼────────────────┐
                        ▼                                 ▼
             ┌──────────────────┐              ┌──────────────────┐
             │  📝 路径 A        │              │  🎤 路径 B        │
             │  简历优化          │              │  模拟面试          │
             │                  │              │                  │
             │  诊断 → 改 → 再诊 │              │  选模式 → 面试    │
             │                  │              │  → 复盘 → 改进    │
             └────────┬─────────┘              └────────┬─────────┘
                      │                                 │
                      │         ┌─────────────────────┐ │
                      └────────▶│   🔄 闭环联动        │◀┘
                                │                     │
                                │ 面试暴露的弱点       │
                                │ → 反馈到简历建议     │
                                │ → 修改简历 → 再面   │
                                └─────────────────────┘
```

## 详细流程

### 阶段 0：首页入口

```
用户打开 https://intervyou.vercel.app

┌──────────────────────────────────────────────────┐
│            🎯 IntervYOU                          │
│          AI 面试教练                               │
│                                                  │
│  校招面试，从简历到面试，我们陪你练到过关            │
│                                                  │
│  ┌──────────────────────────────────────────┐   │
│  │  📎 拖拽或点击上传简历                      │   │
│  │  支持 PDF / Word / Markdown               │   │
│  │                              [开始分析]   │   │
│  └──────────────────────────────────────────┘   │
│                                                  │
│  上传后可以选择：                                  │
│  📝 简历诊断  →  看看你的简历有哪些硬伤             │
│  🎤 模拟面试  →  直接开始练习面试                  │
└──────────────────────────────────────────────────┘
```

**决策点**：用户必须先上传简历。简历是整个系统的唯一信息源。

### 路径 A：简历优化

```
用户点击 "📝 简历诊断"
        │
        ▼
┌──────────────────────────────────────────────┐
│  Step A1: 调用诊断引擎                         │
│                                               │
│  GET /api/resumes/[id]/diagnose               │
│                                               │
│  Claude Sonnet (temperature=0.3)              │
│  System Prompt: 你是资深简历顾问               │
│  输入: 简历结构化 JSON                          │
│                                               │
│  分析 7 个维度：                                │
│  ┌───────────────────────────────────┐        │
│  │ 结构完整性 → 各模块是否齐全          │        │
│  │ 量化密度   → 数字/百分比的密度        │        │
│  │ STAR覆盖  → Situation-Task-Action-Result │
│  │ 亮点突出度 → 核心信息是否在前30%       │        │
│  │ 语言精准度 → 废话/模糊词/被动语态      │        │
│  │ 真实性风险 → 哪些描述容易被追问        │        │
│  │ ATS关键词  → (可选)对标JD的关键词覆盖  │        │
│  └───────────────────────────────────┘        │
│               │                               │
│               ▼ 返回 DiagnosisReport            │
└──────────────┬───────────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────────┐
│  前端：诊断报告展示                              │
│                                               │
│  📝 简历诊断报告                                │
│  ═══════════════════════════════════          │
│                                               │
│  📈 综合评分：72/100                            │
│                                               │
│  🔴 高风险 — 面试中可能被追问致死：               │
│    ⚠ "从0到1搭建AB实验平台"                     │
│      → 如果只能说出大概思路，会被追问致死          │
│      → 建议 A) 补充技术细节 或 B) 改为"参与搭建"  │
│                                               │
│    ⚠ "精通Python"                              │
│      → 会被问装饰器/元类/GIL                    │
│      → 建议改为"熟练掌握，能用于数据分析"         │
│                                               │
│  💡 量化机会：                                  │
│    原文：负责用户增长策略的制定和落地              │
│    改写：主导3个用户增长实验，DAU提升15%          │
│                                               │
│  ┌──────────────────────────────────┐         │
│  │  [按建议修改简历]   [跳过，开始面试]  │         │
│  └──────────────────────────────────┘         │
└──────────────────────────────────────────────┘
               │
     用户选择 "按建议修改简历"
               │
               ▼
      用户在本地修改简历文件
               │
               ▼
      重新上传 → 重新诊断 → 评分是否提升？
               │
               ▼
      满意后 → 进入面试
```

**注意**：MVP 阶段不需要内置简历编辑器（太重），用户在自己电脑上改完重新上传即可。V2 可以加在线编辑。

**详细：每一轮 Agent 内部在做什么？**

```
POST /api/sessions/[id]/chat
Body: { content: "我负责优化推荐策略，引入协同过滤..." }

        │
        ▼
┌─────────────────────────────────────────────────────────┐
│  API Route 处理过程                                      │
│                                                         │
│  1. 从数据库加载上下文：                                   │
│     - 简历结构化数据                                       │
│     - 面试模式 + 难度                                     │
│     - 当前阶段 (phase)                                    │
│     - 问题序号 (questionIndex)                            │
│     - 历史消息 (所有 role=interviewer/candidate 的记录)     │
│                                                         │
│  2. 组装 messages 数组发给 Claude:                        │
│     [                                                    │
│       { role: "user", content: "面试即将开始..." },       │
│       { role: "assistant", content: "{开场白 JSON}" },    │
│       { role: "user", content: "我叫张三，清华..." },      │
│       { role: "assistant", content: "{追问 JSON}" },     │
│       ...                                                │
│       { role: "user", content: "我负责优化推荐策略..." }   │
│     ]                                                    │
│                                                         │
│  3. 调用 Claude API (流式或非流式)                         │
│                                                         │
│  4. Claude 返回：                                         │
│     {                                                    │
│       "message": "协同过滤的冷启动你们怎么处理的？",        │
│       "metadata": {                                      │
│         "evaluation": {                                  │
│           "starCompleteness": 3,                         │
│           "quantification": 2,  ← "没给具体数字"          │
│           "logicClarity": 4,                             │
│           "technicalDepth": 3,                           │
│           "communication": 4,                            │
│           "notes": "技术理解正确但缺量化数据",             │
│           "followUpNeeded": true,                        │
│           "followUpReason": "冷启动是协同过滤的核心难点"   │
│         },                                               │
│         "phase": "resume_deep",                          │
│         "questionIndex": 4                               │
│       },                                                 │
│       "isComplete": false                                │
│     }                                                    │
│                                                         │
│  5. 把消息存入数据库：                                     │
│     - 候选人的回答 (role=CANDIDATE)                       │
│     - 面试官的回复 (role=INTERVIEWER + metadata)          │
│                                                         │
│  6. 返回给前端：                                          │
│     { message, phase, isComplete }                       │
│     (metadata 不返回给前端，面试结束后才用于复盘)           │
└─────────────────────────────────────────────────────────┘
```

#### B4：面试结束判断

```
面试什么时候结束？

┌────────────────────────────────────────┐
│ 条件 1: Claude 主动标记 isComplete=true │
│   触发场景：                             │
│   - 综合模式走完了全部阶段               │
│   - 关键经历都已深挖完毕                 │
│   - 候选人连续3轮回答质量极低            │
│                                        │
│ 条件 2: 轮次达到上限                     │
│   - 简历深挖：12轮                      │
│   - 行为面试：8轮                       │
│   - 技术面试：10轮                      │
│   - 综合模拟：15轮                      │
│                                        │
│ 条件 3: 用户手动结束                     │
│   - 点击 "结束面试" 按钮                 │
│   - 弹窗确认："确定结束吗？将生成复盘报告" │
└────────────────────────────────────────┘
```

## API 调用次数统计（一次完整使用）

假设用户走完整路径：上传 → 诊断 → 修改 → 再上传 → 综合面试(12轮) → 复盘

| 步骤 | API 端点 | 调用 Claude？ | 模型 | Token 估算 |
|------|---------|-------------|------|-----------|
| 上传简历 | POST /resumes | ✅ 1次 | Haiku | ~500 input |
| 简历诊断 | GET /diagnose | ✅ 1次 | Sonnet | ~2000 input |
| 修改后上传 | POST /resumes | ✅ 1次 | Haiku | ~500 input |
| 创建面试 | POST /sessions | ✅ 1次 | Sonnet | ~3000 input |
| 面试 12 轮 | POST /chat ×12 | ✅ 12次 | Sonnet | ~4000 input/轮 |
| 生成复盘 | GET /feedback | ✅ 1次 | Sonnet | ~6000 input |
| **合计** | — | **17次** | — | **~6万 input tokens** |

**成本估算（Sonnet $3/M input tokens）：约 $0.18/次完整面试。** 每天练 100 次也就 $18，非常经济。
