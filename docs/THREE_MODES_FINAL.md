# 三种面试模式 Prompt 设计（最终版）

---

## 一、三层控制体系

三种模式之间不是简单的文字不同，而是**三层控制**：

```
第 1 层：Prompt 指令层（告诉 Agent 做什么）
    ├── 模式定义（模块 6）
    ├── 开场规则（模块 9）
    └── 黑白名单（行为面试专属）

第 2 层：数据控制层（控制 Agent 看到什么）
    ├── 简历深挖/综合：注入完整简历 JSON（含项目描述+亮点）
    └── 行为面试：注入精简简历 JSON（只含公司名+角色，无项目细节）

第 3 层：服务端兜底层
    ├── 重复检测拦截
    └── 换题语替换
```

**之前一直修不好行为面试的原因**：只改第 1 层（Prompt 文字），没动第 2 层（数据）。DeepSeek 看到简历里有"接口延迟从 200ms 降到 80ms"这种数据，不管 Prompt 怎么禁止，它都会去追问技术细节。**删掉数据源，Agent 没东西可问了，自然就只能问行为问题。**

## 二、三种模式的完整对比

### 简历深挖

```
数据层：完整简历 JSON（含项目描述、亮点、技术栈）
指令层：项目深挖型面试官，逐段四层递进
开场：跳过自我介绍，直接点名公司/项目
```

**Agent 看到的简历**：
```json
{
  "experiences": [{
    "company": "字节跳动", "role": "后端开发实习生",
    "highlights": ["接口延迟从200ms降至80ms", "从0到1搭建AB实验平台"],
    "description": "负责抖音电商推荐系统的后端开发..."
  }]
}
```

**第一轮问什么**："我看你在字节跳动做过推荐系统优化，聊聊这个吧——你具体负责了哪些模块？"

---

### 行为面试

```
数据层：精简简历 JSON（只含公司名+角色，无项目描述）
指令层：行为面试官，只能问软技能，有黑白名单
开场：先自我介绍，然后立刻问行为问题
```

**Agent 看到的简历**：
```json
{
  "experiences": [{
    "company": "字节跳动",
    "role": "后端开发实习生"
  }]
}
```

**第一轮问什么**："你在字节跳动实习时，有没有和同事意见不一致的情况？怎么处理的？"

---

### 综合模拟

```
数据层：完整简历 JSON
指令层：真实面试官，严格 7 阶段流程
开场：寒暄+请自我介绍
```

**第一轮问什么**："你好，我是李昂。面试大概15分钟，先从自我介绍开始吧。"

**后续流转**：自我介绍 → 项目深挖（3-5轮）→ 行为问题（2-3轮）→ 情景假设（1-2轮）→ 反问 → 结束

## 三、行为面试的数据控制（本次修复核心）

`system-v2.ts` 第 298 行的条件逻辑：

```typescript
const isBehavioral = interviewMode === 'behavioral';

// 行为面试：简历只保留公司名+角色
// 简历深挖/综合：注入完整简历
...isBehavioral ? {} : { highlights, description }
```

行为面试的 System Prompt 从 ~1400 tokens 降到了 ~800 tokens——砍掉的那部分正是 Agent 之前被诱惑去追问的技术细节。
