# IntervYOU 测试指南

> 确保 `npm run dev` 已启动 → http://localhost:3000

---

## 准备：测试简历

使用 [tests/sample-resume.md](sample-resume.md)，或你自己的简历。

---

## 测试 1：上传简历 + 诊断

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1.1 | 打开首页，拖入 sample-resume.md | 显示文件名 |
| 1.2 | 粘贴一段 JD（可选） | - |
| 1.3 | 点击「开始分析」 | 显示"正在解析简历..." |
| 1.4 | 等待 3-5 秒 | 显示诊断报告：综合评分 + 面试风险 + 改写建议 |

**检查点**：
- [ ] 诊断报告有评分（不是 68 分就好）
- [ ] 「面试风险」指出了简历的具体问题
- [ ] 「改写建议」给出了具体改写示例
- [ ] 底部「诊断记录」有记录

---

## 测试 2：模拟面试

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 2.1 | 在诊断结果页点「开始模拟面试」 | 跳转面试配置页，简历已选中 |
| 2.2 | 选择「简历深挖」模式 + 「标准」难度 | - |
| 2.3 | 点击「开始面试」 | 面试官发开场白 |
| 2.4 | 回答问题时**尽量具体** | Agent 追问更深层问题 |
| 2.5 | 某轮故意回答"不太清楚" | Agent 换题，不继续追问 |
| 2.6 | 点右上角「退出」→「暂时退出」 | 跳转面试记录页，显示「继续面试」 |
| 2.7 | 点击继续，完成面试 | 面试官结束语 → 「查看复盘报告」按钮 |

**检查点**：
- [ ] Agent 开场提到了简历中的公司名（字节跳动）
- [ ] 追问带具体方向，不是"展开说说"
- [ ] 压力模式有明显犀利感（如选压力）
- [ ] 「暂时退出」后能继续

---

## 测试 3：语音输入

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 3.1 | 面试中输入框左侧应有麦克风按钮（Chrome/Edge） | - |
| 3.2 | 点击麦克风开始录音 | 按钮变红脉冲 |
| 3.3 | 说一段话 | 文字实时出现在输入框 |
| 3.4 | 再点击停止 | 可编辑后发送 |

**检查点**：
- [ ] Chrome/Edge 有麦克风按钮，Firefox 没有
- [ ] 转录准确度可接受

---

## 测试 4：复盘报告 + 雷达图

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 4.1 | 面试结束后点击「查看复盘报告」 | 显示报告页 |
| 4.2 | 查看顶部 | 雷达图（五边形）+ 下方分数列表 |
| 4.3 | 向下滚动 | 优势 / 待改进 / 简历建议 |

**检查点**：
- [ ] 雷达图正常渲染
- [ ] 总分不为 72（应有变化）
- [ ] 逐题点评有具体内容

---

## 测试 5：优化简历

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 5.1 | 在复盘报告页点「去优化简历」 | 跳转优化页，自动选综合模式 |
| 5.2 | 选择「仅面试」模式 | 出现面试记录下拉 |
| 5.3 | 选「仅诊断」 | 显示诊断状态 |
| 5.4 | 点「生成优化简历」 | 显示预览 |
| 5.5 | 点「下载 .docx」 | 下载真实 .docx 文件，Word 可打开 |
| 5.6 | 点「复制」 | 复制成功提示 |

**检查点**：
- [ ] .docx 可用 Word 正常打开
- [ ] 优化后的简历量化更多、动词更强
- [ ] 可以切换数据源重新生成

## 测试 6：API Key 配置

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 6.1 | 侧边栏底部「API 配置」 | 展开面板 |
| 6.2 | 填入你自己的 DeepSeek Key | - |
| 6.3 | 点保存 | 显示绿点 + provider 名 |
| 6.4 | 进行一轮面试 | 用的是你自己的 Key（费用从你账户扣） |
| 6.5 | 清除 Key | 恢复使用服务端 Key |
