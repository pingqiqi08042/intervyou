# 面试对话示例

> 场景：简历深挖模式 · 标准难度 · 候选人简历含字节实习（推荐系统优化）

---

## 正常对话应如下

**Agent（开场）**：
张同学你好，我是今天面试官李昂。我们直接开始吧。先聊聊你在字节跳动的实习，你提到优化了抖音电商推荐链路，具体做了什么？

**候选人**：
我们主要是优化了推荐接口的延迟，从 200ms 降到了 80ms。主要做了三件事：用 Kafka 做异步解耦、加 Redis 多级缓存、还有数据库索引优化。

**Agent**：
异步化这块我比较好奇，你们原本是同步调下游服务的，改成异步后下游失败了怎么处理的？有没有数据不一致的情况？

> ✅ Agent 追问了具体技术细节，方向合理

---

**候选人**：
我们做了消息重试机制，如果下游失败了会重试三次。实在不行就走兜底策略，用默认推荐结果。

**Agent**：
了解了。那你简历里还写了从0到1搭建AB实验平台，这个能展开讲讲吗？

> ✅ Agent 觉得异步话题已挖够，自然过渡到下一个亮点

---

**候选人**（如果答不上来）：
呃…AB平台其实主要是另一个同学做的，我主要是用这个平台看数据…

**Agent**：
好的，了解了。那回到推荐接口优化上，你说延迟降到了80ms，这个数字怎么验证的？有做压测吗？

> ✅ Agent 按标准模式：不嘲讽，不纠缠，换到同一个实习的另一个方向

---

## 如果 Agent 只说"请继续"——这是 bug，不是正常行为

出现"请继续"说明 Agent 的 JSON 输出解析失败了。我们已修复：
1. 删除了初始化时的 `__init__` 假请求（会破坏对话历史）
2. 增强了解析出错时的降级逻辑

---

## 测试要点

| 观察点 | 正常表现 | Bug 表现 |
|--------|---------|---------|
| 开场 | 提到具体公司/项目（如"字节跳动"） | 泛泛"请自我介绍" |
| 追问 | 针对你刚说的内容深入问 | "请继续"或重复之前的问题 |
| 换题 | "了解了。你简历上还写了XX..." | 突然跳到完全不相关的问题 |
| 答不上来时 | 记录 → 换题 | 一直追问同一个点 / "请继续" |
